Shell Scenario Panel - Model-Mediated Guardrails Proposal

Date: 2026-01-14 | Scope: Shell Scenario Panel workflow and prompts

Context

The system is intentionally model-mediated and oriented to narrative scaffolding under Knightian uncertainty. The goal is not predictive accuracy but structured sensemaking that expands user thinking. Guardrails should therefore be implemented inside the model workflow (prompts, required sections, explicit self-audit), not via heuristics or regex gates.

Design intent: Make mirroring explicit and bounded, add an antagonistic pass for reflective stress testing, and avoid false precision.

Principles

  • Model-mediated first: Controls live in prompts and required sections.
  • Mirror is a feature: Reflection is useful when it is explicit and bounded.
  • Antagonistic prompting: Every phase includes a structured opposition pass.
  • Provenance over certainty: Inputs are logged as claims with confidence.
  • Discussion over prediction: Outputs bias toward usable narratives and decisions.

Proposed Changes

1) Stance and Uncertainty Class Ritual

At scenario start, the moderator states the stance (mirror, hybrid, analytic) and uncertainty class (Type 1 vs Type 2). This becomes visible in outputs to prevent accidental drift into a predictive tone.

  • Add to shell-scenario-panel/prompts/moderator.md.
  • Echo in phase outputs and executive summaries.

2) Mirror / Opposition / Synthesis Structure

Each phase synthesis explicitly separates faithful reflection from antagonistic counter-frames, then reconciles into a short synthesis. This preserves narrative value while forcing reflective stress testing. 

## Mirror (What the inputs say)
- ...

## Opposition (Plausible alternatives and disconfirmers)
- ...

## Synthesis (What we keep, what we suspend)
- ...
  • Add to shell-scenario-panel/templates/scenario_narrative.md.
  • Use in Phase 2-6 synthesis sections.

3) Provenance and Confidence for New Inputs

When new intel is injected mid-process, capture provenance and falsifiers so the system treats it as a claim, not a fact. This preserves value without uncritical mirroring.

  • Extend shell-scenario-panel/templates/scenario_context_template.md with:
- Source type (open source, proprietary, expert, hearsay)
- Confidence (low, medium, high)
- Corroboration (none, partial, strong)
- Recency (hours, days, weeks)
- What would falsify this?

4) Antagonistic Pass in Specialist Prompts

Each specialist must include a short disconfirming section: alternative hypotheses, failure modes, or reasons their own analysis could be wrong.

  • Add to shell-scenario-panel/prompts/specialists/*.md.
  • Make it a required subsection in transcripts.

5) Quality Analyst: Detect False Precision

Expand the quality gate to flag precision without provenance and synthesis that collapses uncertainty. This is a model-mediated audit, not a hard gate.

  • Update shell-scenario-panel/prompts/specialists/quality_analyst.md.
  • Red flags: single-source dominance, no disconfirmers, tight probabilities on weak data.

6) Near-Term Scenario Constraint

For sub-30-day horizons, require explicit data freshness and uncertainty envelope. This reduces "timeline hallucination" while still allowing narrative detail.

  • Add a short checklist to shell-scenario-panel/prompts/moderator.md.
  • Include a "data freshness" note in outputs.

7) Baseline vs Intel Delta Branching

Run a baseline analysis from open-source inputs, then apply an intel delta memo. This makes the impact of privileged data explicit rather than blended.

  • Document as a small workflow note.
  • Optionally produce a comparison table in reports.

8) Discussion-First Wrap

End outputs with 3-5 prompts that drive discussion rather than prediction (e.g., "What would change your mind?" "What options should stay open?").

  • Add a closing block to shell-scenario-panel/templates/scenario_narrative.md.

Acceptance Criteria

  • Every phase synthesis includes Mirror, Opposition, and Synthesis blocks.
  • Every specialist transcript includes at least one disconfirming alternative.
  • All injected intel entries include provenance, confidence, and falsifiers.
  • Quality audits flag false precision and single-source dominance.
  • Near-term runs state data freshness and uncertainty class.

Note: This proposal preserves the model-mediated philosophy while adding structured antagonistic prompting and transparency guardrails.
